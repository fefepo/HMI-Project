## 📊 실시간 모니터링 HMI
Qt 프레임워크와 C++를 활용하여 장비의 주요 센서 데이터를 실시간으로 모니터링하고, 
위기 상황 발생 시 장비를 보호하는 **인터락 시스템 및 시뮬레이션 HMI**를 구현한 프로젝트입니다.

## 🎯 프로젝트 목표
- 실제 산업 현장에서 사용되는 HMI의 데이터 흐름 및 모니터링 구조 이해
- 실시간 성능을 보장하는 이벤트 기반 프로그래밍(Qt Signal/Slot) 숙달
- 장비와 작업자의 안전을 보장하는 인터락 로직 설계 경험 축적

## 👨‍💻 담당 역할
- Qt 기반 HMI 전체 UI 설계 및 위젯 레이아웃 구성
- TCP/IP 기반 Server/Client 통신 기능 구현
- 실시간 센서 데이터 시각화(QtCharts) 및 업데이트 처리
- 사용자 조작을 통한 장비 상태 시뮬레이션(수동 제어 모드) 개발
- 다단계 안전 인터락(Warning / Emergency Stop) 로직 설계
- 외부 CSV 데이터 파싱 및 시계열 기반 실시간 데이터 재생(Playback) 기능 구현
- 실시간 센서 데이터 및 상태 정보 CSV 로그 저장 기능 구현

## 🔧 사용 기술
- **Language**: C++
- **Framework**: Qt 6
- **UI**: Qt Widgets (HMI UI 구현)
- **Charting**: QtCharts (실시간 데이터 시각화)
- **Network**: Qt Network (TCP/IP Client-Server 통신)
- **Event Handling**: QTimer, Signal/Slot (QTimer, Signal/Slot 기반 비동기 이벤트 처리)
- **Data Logging**: CSV, File I/O (CSV 데이터 로깅 및 파일 읽기) 


## 📌 주요 기능
### **1. TCP/IP 기반 데이터 통신 (Client-Server)**
- **장비 서버(PC 기반 시뮬레이터)**
PC 내부 변수 및 CSV 데이터를 기반으로 장비 동작을 모델링하고 데이터 제공
- **HMI 클라이언트**
Loopback 환경에서 서버에 접속하여 TCP 통신으로 실시간 센서 값(압력, 진동, 온도 등) 수신 및 UI 반영

---

### **2. 실시간 데이터 시각화 (Real-time Charting)**
- **4가지 핵심 지표**(압력 X/Y/Z, 가속도) 및 **파생 데이터**(진동, 온도) 실시간 차트 표시
- **다중 데이터 시계열 분석**: 여러 센서 값을 하나의 차트에서 동시 모니터링
- **가변 시간축 적용**: 데이터 축적에 따른 자동 스크롤 기능

--- 

### **3. 하이브리드 데이터 입력 시스템**
- **Auto Mode**: 외부 CSV 파일로부터 데이터를 읽어와 실시간 차트에 반영(재생)
- **Manual Mode**: HMI 슬라이더 조작 시 해당 센서를 즉시 수동 제어로 전환
- **현실적 시뮬레이션**: 수동 입력 시 실제 환경을 반영한 ±0.5 랜덤 노이즈 적용

---

### **4. 다단계 안전 인터락 (Interlock System)**
- **Warning**: 임계치 초과 시 알림 및 UI 상태 변화
- **Emergency Stop**: 치명적 임계치 초과 시 타이머 중단 및 장비 정지 및 원인 분석 팝업 출력
- **경고/인터락 조건**
  - Warning: 진동 $\ge$ 80.0 / 온도 $\ge$ 70.0
  - Interlock: 진동 $\ge$ 95.0 / 온도 $\ge$ 85.0 / 개별 센서 압력 $\ge$ 90.0

---

### **5. 데이터 로깅 및 분석**
- **실시간 CSV 저장**: 모든 센서 수치와 장비 상태를 타임스탬프와 함께 기록하여 분석 데이터 확보
- **시스템 초기화**: 비상 정지 후 초기화 및 서버 재연결 기능


## 🧠 검사 및 제어 순서
1. 초기화: CSV 로그 파일 생성 및 TCP 서버/클라이언트 소켓 연결
2. 샘플링(20ms): CSV 데이터 또는 사용자 입력값 수집
3. 데이터 가공: 센서 입력값 기반 진동 및 온도 파생값 계산
4. 상태 판정: 인터락 로직을 통해 Warning 또는 Emergency Stop 판단
5. 결과 반영: 차트 시각화 업데이트, UI 상태 갱신 및 센서 데이터 CSV 로그 저장
